关键词：图书馆座位预约；Spring Boot；Vue.js；信用积分；前后端分离AbstractNowadays, the self-study seat resources in university libraries are becoming increasingly scarce. The traditional manual supervision and on-site registration management methods can no longer meet students' demands for study spaces. Long-standing issues such as malicious seat occupation, wasted empty seats, and the time-consuming nature of manual management have consistently posed significant challenges to the daily operations of libraries.

This paper designs and develops an online library seat reservation management platform based on the browser-server architecture, aiming to allocate seat resources rationally through digital and informatized methods, thereby enhancing the library's service capabilities and management efficiency as a whole.

The entire system adopts a frontend-backend separation development model. The backend uses Spring Boot as the core framework, paired with MyBatis Plus for database data operations. The frontend leverages the Vue framework to build separate interfaces for student users and backend administrators. The system fully implements core business functions such as online reservations, QR code check-ins, credit management, temporary seat occupancy, and multi-dimensional data statistical analysis. Additionally, it features a comprehensive rule engine for violation penalties, supporting dynamic configuration of deduction and suspension measures for different violation types. A dual-channel dispute resolution mechanism for appeals and complaints is designed, allowing students to challenge penalty decisions or report misconduct. Integrated feedback management and notification modules enable bidirectional communication between the admin and student interfaces. Real-time seat monitoring views and data export functions further enhance system observability and operational convenience. Through a configurable reservation rule system, administrators can flexibly adjust business parameters without code modifications. The system also introduces a time-decay-based credit calculation method, encouraging proper seat usage with point rewards while deterring misconduct through penalty deductions, gradually guiding students toward rational reservation practices. Extensive platform testing has been conducted, demonstrating stable and reliable system performance that effectively prevents conflicts arising from two students booking the same seat simultaneously. This significantly improves the cyclic utilization efficiency of library seats, making daily management more intelligent and standardized.

以往图书馆大多采用传统人工管理模式，只能让学生自行找座、随意就坐，这种粗放的管理方式暴露出不少弊端。最突出的就是恶意占座问题，不少同学用书籍、水杯等物品长期霸占座位却并不到场学习，既浪费了有限的公共座位资源，也造成了使用上的不公。同时，由于没有统一的座位信息公示，学生只能在馆内来回寻找空位，耗费大量时间精力，学习体验和效率都大打折扣。除此之外，管理人员也没办法实时掌握每个座位的实际使用状态，遇到占座纠纷也没有可靠数据作为依据，整体管理工作量大、效率偏低。

现如今移动互联网、物联网和云计算等技术发展愈发成熟，用信息化方式改造图书馆座位管理模式已经成为必然方向。搭建线上座位预约平台，把线下实体座位转为线上可调度的数字资源，形成预约、签到、自动释放的完整管理流程，既能从根源上减少恶意占座乱象，也能依托数据统计为图书馆优化资源配置提供参考。因此，设计并开发一套功能完备、流程规范、操作便捷的图书馆座位预约系统，有着很强的实际应用价值和现实意义。

1.2 研究目的与意义本次研究希望搭建一套高效公平、便于使用的图书馆座位预约平台，主要目的是合理调配座位资源、规范学生使用行为，同时整体提升图书馆日常管理效率。依靠线上预约方式按需分配座位，能够有效提高现有座位的实际使用率。同时加入信用积分规则，根据用户预约履约情况进行打分约束，慢慢引导大家养成规范用座的习惯。再通过系统完成签到核验、爽约判定和空闲座位释放等流程，减少管理员的人工工作量。

从理论层面看，本系统探讨了信用约束机制在校园公共资源分配中的应用路径，也为共享服务理念在校园场景中的落地提供了一个实践案例。就使用效果而言，它能有效缓解学生自习找座难的问题，减少因抢座引发的种种矛盾。对图书馆管理人员来说，系统给出的数据统计界面直观易用，可以为智慧图书馆的建设提供支撑。投入使用后，这套系统还将带动高校后勤管理的数字化转型，进而提升校园整体的信息化服务水平。

1.3 国内外研究现状1.3.1 国内研究现状在国内，很多学者都围绕图书馆座位管理这一问题，从不同角度展开了研究，研究成果涉及技术怎么落地、管理思路如何优化、相关机制怎样完善等多个方面，也为后续研发座位预约系统，提供了比较实用的参考。

先从技术应用这个方向来看，张敏[1]就拿辽宁大学图书馆当作实际案例，把人脸识别技术融入到入馆选座的整个流程里，不仅能实时监控座位的使用状态，还让图书馆座位管理的智能化程度有了明显提升。布瓦吉尔·艾力[2]则更偏向于用高科技手段赋能管理，他觉得可以通过强化数据统计分析、优化现有系统设计、创新功能搭建框架、简化操作流程这几个方向，打造出既智能又贴合学生使用习惯的图书馆座位预约服务体系。

再说到管理机制和软性优化策略，杨维敏[3]通过实际调研和实证分析，发现除了技术手段，非技术因素对座位管理的影响格外关键。他提出了一套三维协同的低成本优化方案，核心是完善馆内管理制度、引导规范读者使用行为、优化图书馆空间布局和配套服务，这套办法能在控制管理成本的前提下，兼顾座位使用效率和公平性，也给座位资源管控提供了全新的思路。除此之外，杨继民、张贤、徐鸿飞[4]等人，还搭建了制度支撑、配套支持、系统建设三位一体的座位预约管理体系，推动传统座位服务模式实现创新，给学生打造了操作简单、配套齐全的座位预约使用体验。

从总体上看，现有研究从技术赋能、管理制度设计、读者行为干预以及整体系统集成等几个方向切入，无论侧重理论探讨还是实际应用，都为国内高校图书馆优化和改进座位管理提供了切实的参考。

1.3.2 国外研究现状国外图书馆自习室的使用管理，会因为不同地区的国情差异，呈现出不同的管理模式，大致能分成三类。

第一类是以英国、法国、德国、瑞典为代表的欧洲国家[5,6]，这些地方图书馆自习室的管理方式很简洁，既不用专门的管理系统，也不设复杂的预约程序。这主要是因为它们人口偏少，公共资源相对宽裕，无论大学生还是普通市民，都能自由进出图书馆自习，基本不存在资源紧张带来的供需矛盾。

第二类是美国，它的教育资源比较丰富，大部分地区的图书馆不用专门管理自习座位，只有纽约这种人口密集的大城市，部分图书馆和高校才配有简单的座位预约功能。Chu W 等（2017）[7]、Chen X （2017）[8]的研究也提到，这里的预约系统没有严格的监管，全靠使用者自觉，系统也不会强制记录，仅简单提供预约服务，功能比较基础。

第三类是日本、韩国这些亚洲国家，和我们国家的情况很像 —— 人口密度大，自习座位也比较紧张，所以这些国家的图书馆，都配有完善的座位预约和管理系统。Jing-Yu HU(2017) [9]、Guo X G(2018) [10]等人在相关研究中都显示，这些预约管理系统会详细记录每一次的预约信息，还会通过身份核验的方式，确保大家规范使用座位，和我们现在对座位管理的需求特别契合，他们的管理经验也很值得我们参考借鉴。

2系统需求分析2.1 可行性分析2.1.1 技术可行性分析在技术选型上，本系统基于目前主流的 Java EE 技术栈进行开发，这一套体系已经相当成熟。后端通过 Spring Boot 简化了项目配置，结合 MyBatis Plus 让数据的增删改查效率更高；前端采用 Vue.js，能保证页面加载快、交互体验比较顺畅。数据库使用 MySQL，日常的大量数据存储与查询也完全够用。由于对这些技术都比较熟悉，整体开发难度可控，技术层面也没有太大风险。

2.1.2 经济可行性分析从经济角度来算，这套系统完全基于开源框架和免费工具搭建，省去了软件版权授权方面的开销。部署时用普通服务器就足够了，日常维护成本也不高。比起直接采购商业版的图书馆管理软件，自主开发的方式不但整体花费更低、更划算，还能根据学校的具体管理需要灵活调整和定制。

3 系统总体设计3.1 技术介绍后端部分采用 Spring Boot 作为核心开发框架，它可以自动完成项目配置，大幅提升开发效率，同时内置了 Tomcat 服务器，让项目部署变得更加简单方便。数据层使用 MyBatis Plus 工具，它在原有 MyBatis 的基础上做了增强，自带常用的 CRUD 方法，能减少 XML 文件的编写，还支持分页功能和代码自动生成，让开发更高效。数据库选用 MySQL，用来存放用户信息、座位信息、预约记录等系统核心业务数据。

前端采用 Vue.js 框架进行开发，支持组件化开发模式，也能实现数据和页面的双向绑定，让界面交互更加流畅。管理端使用 Element UI 组件库，移动端使用 Vant UI，两者都提供了大量现成的页面组件，能保证整体界面风格统一。前后端的数据交互则通过 Axios 工具完成，用来发送请求和接收返回结果。

3.2 系统总体功能设计本系统整体功能分成三大部分，分别对应不同使用角色。移动端给学生使用，支持账号注册登录、在线约座、扫码签到、中途暂离和签退，还能查看自己的预约记录和个人积分。图书馆管理端供馆内工作人员日常运维，可对图书室、座位信息进行维护，处理预约审核、现场扫码核验，同时负责公告和轮播内容的发布管理。系统管理端由系统理员操作，主要用来统一管理全站用户，查看全体用户的积分记录。各个功能模块依靠 RESTful 接口互相传数据，模块之间互不干扰、独立开发维护，结构松散耦合、内部功能集中，保障系统平稳运行。系统整体功能结构如图3-1所示。

4 操作日志模块设计操作日志审计模块基于Spring框架的AOP（面向切面）编程技术实现。开发人员预先在需要审计的关键Controller方法上标注自定义的@Log注解，注解参数指定操作模块和操作类型。当这些方法被调用时，AOP切面会自动拦截执行，以异步方式提取以下信息并写入数据库：当前登录用户的身份标识（ID和用户名）、操作所属模块、操作类型（新增/修改/删除/导出/审核等）、操作对象的类型和ID、客户端请求IP地址、浏览器User-Agent标识、以及方法的输入参数和返回结果的JSON摘要。

1 信用积分机制设计系统设计了一套与学生学号绑定的信用积分管理制度，全程记录用户的座位使用行为，系统会按照规则自动更新积分，并依据积分分数为用户提供对应的预约权限。学生首次注册账号时，系统会自动赋予80分的初始积分。积分核算采用时间衰减算法，近期的使用行为对积分影响更大，以往的违约记录可通过后续良好表现逐步抵消。积分加分规则：按时签到使用座位加 1 分，提前结束使用并签退加 1 分，每天最多可获得 2 分加分。积分扣分规则：预约后未到场使用扣 6 分，临时离开座位超时未返回扣 3 分，使用结束后未签退直接离开扣 3 分。系统将信用积分划分为五个等级，分别为：≥90 分（极好）、75-89 分（良好）、60-74 分（一般）、40-59 分（较差）、低于 40 分（暂停服务）。不同积分等级对应不同的预约时长和每日可预约次数。该模块的功能设计流程图如图4-22所示。

2 预约冲突检测机制为保证座位资源合理分配，系统采用双重校验方式完成预约冲突检测。用户提交预约申请时，第一步进行座位时段冲突校验，通过SQL语句查询目标座位在所选时间段内是否存在已预约、已使用的状态，只要新预约时段与已有记录存在时间重叠，系统都会判定为冲突。第二步进行用户自身预约校验，避免同一学生在同一时间段预约多个座位。所有预约相关操作都在事务中完成，借助数据库锁机制保障高并发场景下的数据安全，从根本上避免重复预约、座位超订等问题。

3 违规检测与自动处罚机制设计本系统的违规检测采用服务端定时任务+规则引擎的双重架构模式。后端通过Spring Task调度器设定定时任务，每隔固定间隔（通常为1分钟）扫描所有活跃的预约记录，依次检查以下违规场景：（1）爽约检测：预约开始时间已过且用户未完成签到操作；（2）暂离超时检测：用户标记暂离后超过规定时长（可配置，默认20分钟）仍未取消暂离返回；（3）未签退离馆检测：预约结束时间已过用户未主动签退。

一旦检测到违规行为，系统进入规则匹配流程：首先统计该用户同类违规的历史发生次数，然后在punishment_rule表中查找违规类型+适用次数匹配且状态为&quot;启用&quot;的规则，读取其中配置的扣分值和封禁天数，自动生成一条violation_record记录（状态设为&quot;待处理&quot;），同时同步执行积分扣减操作。整个检测过程全自动运行，无需人工干预，但处罚执行前留有人工审核环节（违规记录状态为&quot;待处理&quot;），管理员确认后才正式生效，兼顾了自动化效率与管理灵活性。该模块的功能设计流程图如图4-24所示。

7结论7.1 总结本次课题完成了高校图书馆自习室线上预约管理系统的开发，针对性解决了校园图书馆传统人工管理模式下的各类弊端。系统借助积分信用机制和后台自动定时任务，既实现了座位资源的合理调配管控，也能约束学生用座行为，形成自主规范的使用习惯。项目整体选用 Spring Boot 加 Vue 前后端分离开发模式，整体结构规整，后续新增功能、日常维护都十分方便。投入使用后，既能提升图书馆座位的循环利用效率，也大幅降低了管理人员的日常工作量。

7.2 展望现阶段系统还有不少可以完善的地方，目前还没接入蓝牙定位技术，无法实现自动无感签到，数据统计分析的角度也不够丰富。后续可以加装物联网感应设备，实时监测每个座位的实际使用情况；同时引入智能算法预判人流高峰时段，优化座位分配方案，进一步完善调度机制，打造更智能化、便民化的校园图书馆服务模式。
